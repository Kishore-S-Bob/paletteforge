1. Product Overview

1.1 Product Vision
PaletteForge is a free, browser-based colour palette generator that lets designers and developers pick a base colour and instantly generate 5 palette types — with one-click export as CSS variables, Tailwind config, or SCSS. Every generated palette gets a unique shareable URL, making the product inherently viral: every share is a free advertisement.

1.2 Problem Statement
Designers and developers need colour palettes constantly, but existing tools (Coolors, Adobe Color) are either slow, require accounts, or produce palettes that cannot be exported in developer-ready formats. There is no single clean tool that combines instant palette generation, WCAG contrast checking, and multi-format code export — all without login or cost.

1.3 Product Goals
• Deliver a palette from a single colour click in under 1 second
• Make every palette shareable via a URL — the core viral loop
• Export palettes in CSS, Tailwind, and SCSS formats developers actually use
• Reach 1,000 daily active users within 30 days of launch with $0 marketing spend
• Generate first revenue within 60 days via Google AdSense

4.2 Architecture Overview
PaletteForge is a fully static Next.js application for Phase 1 and 2 — no server, no database, no backend. All colour generation happens in the browser using chroma.js. The shareable URL system uses query parameters only — no server-side storage required. This keeps running costs at zero and page load times under 1 second globally.

Phase 3 introduces optional Supabase integration for user accounts and saved palettes, but the core tool remains fully functional without login.

5. UX & Design Requirements

5.1 Design Principles
• Screenshot-first — every palette row must look beautiful in a social media screenshot
• Zero friction — no login, no account, no popups on first visit
• One screen — the entire tool should be visible without scrolling on desktop
• Copy-ready — every piece of output (hex, code, URL) copies in one click

5.2 Page Structure
SectionContentHero SectionTool name, one-line description, colour picker prominently displayedPalette Display5 palette type rows, each with 5 colour swatches + hex labelsExport Panel3 tabs: CSS / Tailwind / SCSS. Code block with copy button.Share Section'Tweet this palette' button + 'Copy link' button side by sideFeatured Gallery12 pre-made palettes in a 4-column grid below the generatorFooterAbout, GitHub link, AdSense unit (Phase 3)
5.3 Colour Swatch Design
• Each swatch: minimum 80px tall, full width divided equally across 5 colours
• Hex code displayed on hover (desktop) and always visible on mobile
• Click copies hex to clipboard — show brief 'Copied!' toast confirmation
• WCAG pass/fail badge overlaid on swatch (Phase 2)

6.3 Growth Loop
The core viral mechanic: every time a user clicks 'Tweet this palette', Twitter shows the palette image and the tool URL to their followers. Designers tweet palettes constantly — this is existing behaviour, not new friction. The goal is to insert PaletteForge into a habit that already exists.

• User generates palette they love
• Clicks 'Tweet this palette' — pre-filled tweet with image + URL
• Their followers see the palette and click the link
• New user generates their own palette and tweets it



7. Monetisation Plan

7.1 Phase 1 — No Revenue (Days 1–30)
Focus entirely on growth. No ads, no paywalls, nothing that slows adoption. The goal is to reach 1,000 daily active users before monetising.

7.2 Phase 2 — AdSense (Days 30–60)
Apply for Google AdSense once daily traffic exceeds 100 visitors. Design and dev audiences command $3–8 RPM. At 5,000 visitors/day that is $15–40 per day passively — approximately $450–1,200 per month.
